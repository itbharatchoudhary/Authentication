# Terminal commands

## Step-1
-> npm init -y

npm init -y – Quickly initializes a new Node.js project by generating a default package.json file without prompting for user input.

## Step-2
-> npm i express mongoose morgan dotenv jsonwebtoken cookie-parser nodemailer axios bcryptjs cors express-rate-limit google-auth-library winston

- **express** – Minimal and flexible Node.js web framework for building APIs and handling HTTP requests efficiently.
- **mongoose** – ODM (Object Data Modeling) library for MongoDB, enabling schema-based data modeling and easy database interaction.
- **morgan** – HTTP request logger middleware for tracking and debugging API requests in development and production.
- **dotenv** – Loads environment variables from a `.env` file into `process.env` for secure configuration management.
- **jsonwebtoken** – Implements JSON Web Tokens (JWT) for secure authentication and authorization mechanisms.
- **cookie-parser** – Middleware to parse and manage cookies attached to client requests.
- **nodemailer** – Module for sending emails (e.g., OTPs, verification links) via SMTP or third-party services.
- **axios** – Promise-based HTTP client for making external API requests (e.g., Google authentication).
- **bcryptjs** – Library for hashing and comparing passwords securely using cryptographic techniques.
- **cors** – Middleware to enable Cross-Origin Resource Sharing, allowing secure API access from different domains.
- **express-rate-limit** – Middleware to limit repeated requests, protecting APIs from brute-force and abuse attacks.
- **google-auth-library** – Official Google library for handling OAuth2 authentication and verifying Google tokens.
- **winston** – Advanced logging library for structured, scalable, and production-grade logging.
